# 操作步骤和指导文档（VS Code + Copilot + Obsidian）

## 1. 核心概念

LLM-Wiki 是一种由 AI 持续维护的知识库框架：

- 原始素材层：raw/，只读，负责保留证据与溯源。
- AI 编译层：wiki/，负责摘要、概念拆分、实体归档与跨文链接。
- 规则配置层：.instructions.md 与模板，约束 AI 的行为一致性。

与传统 RAG 相比，LLM-Wiki 的重点是把知识沉淀为可持续维护的页面，而不是每次查询都从原始文档重新检索。

## 4. 一次性初始化步骤（首日）

1. 在 Obsidian 中打开此仓库作为 Vault。
2. 在浏览器安装并配置 Web Clipper，默认保存目录为 raw/clippings/。
3. 在 VS Code 中打开 Copilot Chat（建议 Agent 模式）。
4. 使用下方初始化提示词，让 Copilot核对结构完整性。

初始化提示词：

```text
请将当前仓库作为 LLM-Wiki 执行环境。要求：
1) 不修改 raw 下任何文件；
2) 以 wiki/index.md 作为导航入口；
3) 每次写入 wiki 内容都要追加 wiki/log.md；
4) 输出中文；
5) 先检查目录结构是否完整，缺失则补齐。
```

## 5. 三大核心操作

### 5.1 Ingest（导入资料）

适用场景：新增文章、网页剪藏、PDF、笔记。

操作：

1. 把资料放入 raw 对应子目录。
2. 发送 Ingest 提示词。

Ingest 提示词：

```text
读取 raw 下新增或更新的文件，按以下规则编译：
- 先在 wiki/sources 生成每个来源的摘要页；
- 再更新相关 wiki/concepts、wiki/entities、wiki/synthesis；
- 维护 See Also 交叉链接；
- 刷新 wiki/index.md 的表格项；
- 在 wiki/log.md 追加本次 ingest 记录（日期、处理文件、更新页面）。
```

### 5.2 Query（查询知识）

适用场景：总结、比较、回顾。

Query 提示词：

```text
请先读取 wiki/index.md，再读取相关页面回答问题。
回答要求：
- 仅基于 wiki 现有页面；
- 使用 markdown 链接给出引用；
- 输出中文、结论优先。
问题：{在这里写你的问题}
```

如需归档答案：

```text
将刚才的回答归档到 outputs/query.md，保留引用链接，并写入归档时间。
```

### 5.3 Lint（巡检）

适用场景：每周维护、批量导入后校验。

Lint 提示词：

```text
请执行 wiki 巡检：
1) 检查 wiki/index.md 条目与实际文件一致性；
2) 检查 wiki 内部死链接；
3) 检查孤立页面与冲突陈述；
4) 自动修复可确定问题；
5) 将报告写入 outputs/lint.md；
6) 在 wiki/log.md 追加 lint 记录。
```

可选命令行巡检（本地脚本）：

```bash
python scripts/lint.py
```

## 6. 借鉴 B 站教程的落地策略

1. 0 代码优先：先用自然语言提示词跑通流程。
2. 轻量起步：先导入 3-5 篇短文，不要一开始灌入大量 PDF。
3. 模板复用：references/ 模板稳定后再扩展领域。
4. 中文优化：统一中文命名和输出风格，减少中英文混排噪声。
5. 可视化反馈：在 Obsidian 图谱查看 concepts 与 entities 的连通性。

## 7. 常见避坑

1. 不要手改 raw 原文，否则难以追溯来源。
2. 规则文件不要频繁大改，每次只改一项并做单篇验证。
3. 每周至少跑一次 Lint，每月做一次仓库备份。
4. AI 自动合成内容要抽检关键结论与引用路径。

## 8. 首轮验收清单

1. raw/clippings 中至少有 1 篇素材。
2. wiki/sources 至少新增 1 个摘要页。
3. wiki/index.md 已出现新增条目。
4. wiki/log.md 已记录一次 ingest。
5. outputs/lint.md 已生成巡检结果。

## 9. Git 同步建议

首次提交：

```bash
git add .
git commit -m "init: bootstrap llm-wiki structure and guide"
git push -u origin main
```

后续建议：

- 每次 ingest 后提交一次，便于回滚与审计。
- 对规则文件改动使用独立提交，避免与知识内容混在一起。
