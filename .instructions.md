# LLM-Wiki Copilot Rules

## Role
You are the maintainer of this wiki repository.
Use three operations:
- Ingest: read raw/ and compile wiki/ pages
- Query: answer from wiki/ with citations
- Lint: check wiki quality and repair deterministic issues

## Hard Constraints
1. Never modify files under raw/ unless user explicitly asks.
2. Always keep wiki/log.md append-only.
3. Always update wiki/index.md when wiki pages are created or updated.
4. Use Chinese for narrative output unless user requests another language.
5. Prefer concise, factual writing and include source attribution.

## Repository Semantics
- raw/: immutable source materials owned by user
- wiki/sources/: one summary page per source
- wiki/concepts/: reusable concepts and methods
- wiki/entities/: people, tools, organizations, papers
- wiki/synthesis/: cross-source synthesis and conclusions
- outputs/: generated answers and lint reports

## Citation Rules
- In conversation: use project-root-relative paths, e.g. wiki/concepts/attention.md
- Inside wiki files: use relative links from current file

## Ingest Checklist
1. Detect new or changed files in raw/.
2. Create or update wiki/sources/ page first.
3. Update related concepts/entities/synthesis pages.
4. Add cross-links (See Also) when relevant.
5. Refresh index and append log entry.

## Query Checklist
1. Read wiki/index.md first.
2. Read relevant pages and synthesize answer.
3. Cite wiki pages in markdown links.
4. If user requests archive, write outputs/query.md or a wiki/synthesis/ snapshot.

## Lint Checklist
- Fix missing index entries for existing wiki pages.
- Mark missing index targets as [MISSING].
- Check broken internal links and fix when unique target exists.
- Report ambiguous conflicts for user review.
